# L2 — Detailed Requirements

## L1-1: Link Command

### L2-1.1: CLI Options for Link

The link command (root command) shall accept `--workspace` (`-w`), `--library` (`-l`), and `--source` (`-s`) options, all required.

**Acceptance Criteria:**
- Invoking the CLI without required options produces a usage error.
- All three options are parsed and forwarded to the link service.

### L2-1.2: Input Validation for Link

The link command shall validate that the workspace path exists and contains `angular.json`, and that the library source path exists and contains a `package.json` with a matching library name.

**Acceptance Criteria:**
- Returns exit code 1 when the workspace path does not exist.
- Returns exit code 1 when the library source path does not exist.
- Returns exit code 1 when `angular.json` is missing from the workspace.
- Returns exit code 1 when `package.json` is missing or has a mismatched name.
- No npm commands are invoked when validation fails.

### L2-1.3: npm link Execution

The link command shall run `npm link` in the library source directory, then `npm link <library>` in the workspace directory.

**Acceptance Criteria:**
- `npm link` is invoked first in the library source directory.
- `npm link <library>` is invoked second in the workspace directory.
- If the first command fails, the second is not executed and the exit code is propagated.
- If the second command fails, its exit code is propagated.
- Returns exit code 0 on success.

### L2-1.4: tsconfig.json Path Update

The link command shall add path mappings for the library in the workspace's `tsconfig.json`.

**Acceptance Criteria:**
- `compilerOptions.paths` receives entries for `<library>` and `<library>/*` pointing to relative paths to the library source.
- If `tsconfig.json` does not exist, the command still succeeds.

## L1-2: Unlink Command

### L2-2.1: CLI Subcommand Registration

The CLI shall register an `unlink` subcommand that accepts the same `--workspace` (`-w`), `--library` (`-l`), and `--source` (`-s`) options as the link command, all required.

**Acceptance Criteria:**
- `npm-link unlink -w <path> -l <name> -s <path>` is a valid invocation.
- Omitting any required option produces a usage error.

### L2-2.2: Input Validation for Unlink

The unlink command shall validate that the workspace path exists and contains `angular.json`.

**Acceptance Criteria:**
- Returns exit code 1 when the workspace path does not exist.
- Returns exit code 1 when `angular.json` is missing from the workspace.
- No npm commands are invoked when validation fails.

### L2-2.3: npm unlink Execution

The unlink command shall run `npm unlink <library>` in the workspace directory, then `npm unlink` in the library source directory.

**Acceptance Criteria:**
- `npm unlink <library>` is invoked first in the workspace directory.
- `npm unlink` is invoked second in the library source directory.
- If the first command fails, the second is not executed and the exit code is propagated.
- If the second command fails, its exit code is propagated.
- Returns exit code 0 on success.

### L2-2.4: tsconfig.json Path Removal

The unlink command shall remove the path mappings for the library from the workspace's `tsconfig.json`.

**Acceptance Criteria:**
- The `<library>` and `<library>/*` keys are removed from `compilerOptions.paths`.
- Other existing path entries are preserved.
- If `tsconfig.json` does not exist, the command still succeeds.
- If the paths object does not contain the library keys, no error occurs.
