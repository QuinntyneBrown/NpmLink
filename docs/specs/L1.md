# L1 — High-Level Requirements

## L1-1: Link Command

The CLI shall provide a link command that symlinks a local npm library into an Angular workspace for local development and debugging.

## L1-2: Unlink Command

The CLI shall provide an unlink command that reverses the linking process, removing the symlink from the Angular workspace, unregistering the library globally, and cleaning up tsconfig.json path mappings.

## L1-3: Verify Command

The CLI shall provide a verify command that checks whether a library is correctly linked to an Angular workspace, reporting the status of each check and returning an appropriate exit code. Verification shall validate both the presence and correctness of tsconfig path mapping values.

## L1-4: JSONC-Safe tsconfig Handling

All tsconfig.json parsing and mutation shall be JSONC-safe, supporting comments and trailing commas. Parse and write failures on present tsconfig files shall cause the operation to fail rather than silently succeed.

## L1-5: Typed npm Process Invocation

npm process execution shall use structured, typed invocations rather than shell string commands. The system shall use `npm.cmd` on Windows and `npm` on non-Windows instead of `cmd /c`.

## L1-6: Dependency Injection and Separation of Concerns

The CLI shall use a DI composition root. Application logic shall be separated from IO and console output through abstractions (`ITsConfigEditor`, `INpmClient`) and structured result types (`OperationResult`).

## L1-7: Comprehensive Test Coverage

The test suite shall cover failure modes including wrong tsconfig mapping values, JSONC content, invalid tsconfig content, missing library source path during unlink, and paths containing spaces.

## L1-8: Clean Tooling

The project shall build and test without warnings. Package versions shall resolve cleanly without NU1603 warnings.
